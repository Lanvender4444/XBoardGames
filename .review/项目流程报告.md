## 1. 项目是什么

一个**可扩展的 AI 桌游引擎平台**。核心目标三条：

1. **规则即数据**：游戏不写死在代码里，而是用 `Rule.md`（YAML front-matter + 正文）描述；
   "喂一份规则文书 → 自动生成可执行游戏"。内置**狼人杀**（可玩）与**阿瓦隆**（结构样本）。
2. **有记忆的 AI 玩家**：AI 人物卡带**短期记忆（心证）+ 长期记忆（跨局）+ 羁绊**，会"记得你上次那波操作"。
3. **双模式联机**：局域网（LAN，本机 sidecar）/ 服务器（集群）两套存储档案，同一套业务代码。

技术栈：后端 Python（引擎纯逻辑 + FastAPI），前端 Tauri + React + TypeScript，
存储抽象层（SQLite/MySQL · Redis · 向量库），AI 决策子图（可接 LangGraph + LLM）。

规模快照：后端 40 个 py 文件 / ~3300 行 + 71 个测试；前端 9 个 ts/tsx；2 份内置 Rule.md。

## 2. 总体架构（分层）

```
┌─────────────────────────────────────────────────────────────┐
│  前端 apps/desktop （Tauri + React + TS，包豪斯 UI）           │
│  大厅 / 规则 / 牌桌 / 角色卡  ── 只发"意图"，渲染服务器下发的合法行动 │
└───────────────▲──────────────────────────┬──────────────────┘
                │ WebSocket / HTTP          │ 共享协议类型
                │                           ▼
        ┌───────┴───────────────  packages/protocol （TS↔Py 单一协议） ┐
                │
┌───────────────▼──────────────────────────────────────────────┐
│  后端 apps/backend                                             │
│                                                               │
│   api (FastAPI/WS)  ──  multiplayer (会话/广播/发现)            │
│        │                      │                                │
│        ▼                      ▼                                │
│   rules (摄取管线)      engine (纯逻辑状态机)  ◀── agents (AI 决策子图) │
│        │                      │                      │         │
│        ▼                      ▼                      ▼         │
│   ── storage 抽象层 (Repository/StateStore/EventBus/VectorStore) ── │
│        │             memory (STM/LTM/羁绊/瞬间/固化)             │
│   local: SQLite+内存/Redis+FAISS   server: MySQL+Redis+pgvector │
└───────────────────────────────────────────────────────────────┘
```

**关键架构约束**：`engine` 是纯逻辑，**不 import 网络/LLM**（有单测守护）。引擎对"行动来自人还是 AI、
本机还是远程"完全无感知——所有来源都变成 `Action` 喂给 `engine.apply`。这条约束让引擎可独立单测、
让联机/AI 成为可插拔的外围。

## 3. 构建历程（按 Phase）

| Phase | 内容 | 状态 | 关键产物 |
|---|---|---|---|
| 0 | 地基：Monorepo 目录、存储抽象层、协议、SQLAlchemy 模型、配置/路径 | ✅ | `core` `storage` `models` |
| 1 | 引擎 + 内置狼人杀（无 AI、无联机） | ✅ 可运行+测试 | `engine` `rules` `cli/autoplay` |
| 2 | AI 玩家（决策子图+启发式策略）+ 记忆（STM/LTM/羁绊/瞬间/固化） | ✅ 已实现+测试 | `agents` `memory` |
| 3 | 联机（会话/广播/房间发现）+ FastAPI/WebSocket | ✅ 已实现+测试 | `multiplayer` `api` |
| 4 | 规则摄取管线（文书 → 结构化 → Rule.md → 编译） | ✅ 已实现+测试 | `rules/parser` |
| 5 | 精彩瞬间 + 记忆衰减 + 打磨 | ✅ 已实现+测试 | `memory/highlights` `ltm.decay` |

> 演进说明：Phase 2–5 最初是接口占位（`NotImplementedError`），现已全部落地为**真实可运行实现**。
> 实现策略——默认用**无外部依赖、确定性的本地实现**跑通整链并单测，同时把 LLM / Redis / 向量库 / OCR
> 放在**可注入接口**之后；生产环境换实现即可、业务代码不变。详见 [detailed/07](detailed/07-phase2-5-implementation.md)。

## 4. 运行时流程（端到端）

### 4.1 规则 → 可玩
```
文书(PDF/docx/txt/md)
  → parser.extract_text                提取纯文本
  → parser.structure_to_rule_md        结构化为 Rule.md 草稿（默认启发式，可注入 LLM）
  → 前端编辑器人工审校 + 实时校验          routes_rules.validate_text
  → compiler.compile_rule_md           编译为 GameDefinition（状态机 + 原语绑定，编译期校验）
  → SessionManager.create_session      开局
```
内置游戏走捷径：`load_builtin("werewolf")` 直接编译仓库里的官方 Rule.md。

### 4.2 一局之内（实时对局，服务器权威）
```
客户端发"意图" Action（前端只点服务器下发的 legal_actions）
  → SessionManager.submit_action
      → engine.apply        校验合法性 + 收集/即时结算，产出带 visibility 的 Event
      → EventBus 广播        按可见性投递（public/private/faction）；StateStore 落快照
  → 轮到 AI 席位 → AIPlayer/HeuristicPolicy.decide
      → DecisionGraph 五节点子图：
          感知 perceive：取 legal_actions、识别狼队友
          召回 recall  ：扫"自己可见的"私有事件 → 更新 STM 心证；取 BondGraph 行为偏置
          推理 reason  ：给每个合法行动打分（狼不刀队友 / 预言家查未知 / 投最可疑）
          行动 act     ：取最高分 → 过 engine 合法性双保险 → 落地
          编码 encode  ：心证写回（跨回合记忆）
      → 产出的 Action 同样回到 engine.apply（人/AI 同一条路径）
  → 全员行动完 → engine.advance_phase（结算+转移+判胜负）→ 广播 phase_changed / game_over
```

### 4.3 一局结束（记忆固化，跨局）
```
game_over
  → consolidate_session(events, seats)
      → HighlightEvaluator.scan          识别神预言/关键一票/极限翻盘 → episodic 记忆
      → 终局身份                          → 每个 AI 对他人的 semantic 印象
      → Embedder 向量化 → LongTermMemoryStore.write（入 VectorStore）
      → BondGraph.apply_outcome           按救/票/夜杀更新羁绊 + 打标签（救过我/宿敌…）
      → 各角色 STM.clear
```

### 4.4 下一局（记忆生效，形成"人设/恩怨"）
```
DecisionGraph.recall
  → LongTermMemoryStore.recall(当前情境)   召回"上局这人是狼 / 那次被坑"
  → BondGraph.to_behavior_bias            对宿敌更针对、对恩人更信任
  → 注入推理打分 → 行为随历史改变
  ↺ LongTermMemoryStore.decay             低价值记忆衰减遗忘，防向量库膨胀
```

## 6. 设计原则与可替换接缝

延续存储抽象层思路，所有重依赖都在协议接口之后，本地默认确定性、生产可替换：

| 接缝 | 本地默认 | 生产可换 |
|---|---|---|
| Embedder | HashingEmbedder（哈希词袋） | 句向量模型（bge / text-embedding-3） |
| Reasoner | HeuristicReasoner（启发式打分） | LangGraph + LLM 推理节点 |
| Structurer | HeuristicStructurer（透传/骨架） | LLM 规则结构化 |
| StateStore / EventBus | 内存实现 | Redis / Redis Pub/Sub |
| VectorStore | InMemoryVectorStore | FAISS(local) / pgvector(server) |
| Repository | InMemoryRepository | SQLAlchemy(SQLite/MySQL) |

其它贯穿全局的原则：服务器权威（客户端只发意图）；`request_action` 必带 `legal_actions`（前后端一致）；
事件带 visibility 按席位投递；可写数据只落用户目录（绝不写安装目录）。

## 7. 怎么运行

```bash
# 后端（引擎 + AI + CLI 对局）
cd apps/backend
uv sync --extra dev --extra storage
uv run pytest                                   # 71 passed
uv run python -m app.cli.autoplay --game werewolf --players 8 --policy heuristic

# 后端 API（联机/规则路由/WebSocket）
uv sync --extra api
uv run uvicorn app.api.app:create_app --factory --host 0.0.0.0 --port 8765

# 前端（包豪斯 UI）
cd apps/desktop
npm install
npm run dev                                     # http://localhost:5173
npm run tauri dev                               # 桌面窗口（需 Rust 工具链）
```

完整启动路径见根 [`README.md`](../README.md)。
