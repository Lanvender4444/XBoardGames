# 前端可玩狼人杀 + 大模型前端可配置 实现报告

> 目标：在前端界面真正玩起来人机狼人杀；大模型 API 在前端可配置，支持所有主流开源/闭源模型。
> 验证：后端 `uv run pytest` → **99 passed**（新增 6 个 play 用例，覆盖建局/出牌/整局/非法行动/LLM 配置脱敏/provider 清单）；
> 前端 `npm run build` 通过（tsc + vite，51 模块）；TestClient 跑通"人类+AI"整局 HTTP 对局。

## 1. 整体链路

```
浏览器(React, 包豪斯)  ──HTTP──>  FastAPI(play API)  ──>  引擎 + LangGraph 决策子图 + LLM 决策链
  设置页配置 LLM ───────────────> /llm/config (运行期 provider 配置)
  大厅建局 ─────────────────────> POST /games
  牌桌出牌 ─────────────────────> POST /games/{id}/action → 服务器自动驱动 AI 到下一次人类回合
```

## 2. 大模型：前端可配置 + 支持所有主流

"支持所有主流"的工程做法：**OpenAI 兼容协议**（`base_url + api_key + model`）几乎是行业事实标准，
绝大多数厂商与本地推理服务都提供它，所以用一个 OpenAI 兼容适配器即可覆盖；Anthropic 走原生适配；
另有内置离线模型作默认（无需 Key）。

`app/agents/providers.py` 预设（共 15 项，前端下拉可选并自动填好 base_url/默认模型）：

| 类型 | 覆盖 |
|---|---|
| 闭源云 | OpenAI、DeepSeek、Moonshot(Kimi)、智谱 GLM、通义千问(DashScope 兼容)、Mistral、Anthropic(原生) |
| 聚合/加速 | OpenRouter、Groq、Together |
| 本地/自建 | Ollama、LM Studio、vLLM（无需 Key） |
| 其它 | offline(内置离线，默认)、custom(自填 base_url) |

前端**设置页**字段：Provider 下拉、Base URL、Model、API Key（密码框、脱敏回显）、Temperature 滑杆、后端地址。
保存 → `POST /llm/config`（key 不落盘、回显脱敏）。之后建的对局，AI 即用该模型。

`build_chat_model(config)`：offline 或缺 key → 内置离线模型；anthropic → `ChatAnthropic`；其余 → `ChatOpenAI(base_url,...)`。
缺对应 langchain 库时自动回退离线，**保证总能运行**。

## 3. 可玩对局 API（`app/api/play.py`）

框架无关的 `PlayService` + 懒加载 FastAPI 路由：

- `POST /games {slug,players,human_seats,seed}` → 建局，立即把"开局到第一个人类回合"之间的 AI 行动跑掉，返回视图。
- `GET /games/{id}?seat=` → 视图：席位（按可见性裁剪他人身份）、阶段/回合、可见事件流、**你的合法行动（带中文标签）**、是否轮到你、等待中的席位、终局胜者。
- `POST /games/{id}/action {seat,type,targets}` → 提交人类行动 → 引擎校验 → **自动驱动 AI**（每个 AI 席位跑 LangGraph+LLM 决策子图）+ 阶段结算，直到再次轮到人类或对局结束 → 回新视图。非法行动 → 400。
- `GET /providers`、`GET/POST /llm/config` → provider 清单与 LLM 配置。

**人机交替的核心**（`_drive`）：循环——无人行动则 `advance_phase`；若当前阶段有人类待行动则停下等输入；否则把所有 AI 行动跑掉；直到终局或轮到人类。

## 4. 前端（`apps/desktop/src`）

- `api.ts`：后端客户端，后端地址可配（localStorage，默认 `http://localhost:8000`）。
- `store/index.ts`：zustand，接 `createGame/act/view` 与 LLM 配置；去掉了旧的模拟数据。
- `features/lobby`：选人数、开始对局、跳设置。
- `features/settings`：LLM provider 配置（下拉自动填 base_url/model）+ 后端地址。
- `features/table`：真实牌桌——席位网格（你高亮、死亡置灰、身份按可见性）、事件流、终局横幅、**你的行动按钮**（点按即 `POST action`，AI 自动接管到下一次你的回合）。
- `App.tsx`：导航 大厅 / 牌桌 / 角色 / 设置；刊头显示当前回合阶段与 LLM provider。

CORS 已在后端开启（前端 5173 与后端 8000 跨源）。

## 5. 怎么跑

```bash
# 后端
cd apps/backend && uv sync --extra api --extra ai
uv run uvicorn "app.api.app:create_app" --factory --port 8000
# 前端
cd apps/desktop && npm install && npm run dev   # http://localhost:5173
```
设置页配模型(或用默认离线) → 大厅开局 → 牌桌开玩。
